# masautt.github.io

A running reference to the things in my life. Built with plain Jekyll (native to
GitHub Pages ‚Äî no build config needed). Live at <https://masautt.github.io>.

## This repo hosts; it doesn't render sections

Each section is built in its own `masautt-inc` repo and **pushed here as static files**
over an SSH deploy key. This repo owns the landing page, the shared layout, and the CSS.

| Path | Built by | What it pushes |
| --- | --- | --- |
| `/devices` | `masautt-inc/devices` | a whole rendered Jekyll site ‚Üí `devices/` |
| `/hotkeys`, `/dev` | `masautt-inc/config` (`site/`) | Jekyll pages that this repo's layout renders |

A push from one of those deploy bots lands on `main` and triggers `pages.yml`, which
builds and publishes.

```
_config.yml              site config (no collections ‚Äî see the warning below)
_layouts/default.html    <head>, CSS link, page wrapper
assets/css/site.css      all styling
index.md                 home / hub landing
devices/                 PUSHED by masautt-inc/devices ‚Äî do not edit
hotkeys/  dev/           PUSHED by masautt-inc/config ‚Äî do not edit
```

> **Don't add a collection whose permalink collides with a pushed section.** This repo used
> to render `/devices/:name/` from a `_devices/` collection. Once `masautt-inc/devices`
> started pushing to those same paths, Jekyll's collection won and silently shadowed the
> pushed pages ‚Äî deploys reported success while four of five device pages served stale
> content for weeks. The collection was removed in the 2026-07 cutover.

## Editing a section

Don't edit `devices/`, `hotkeys/`, or `dev/` here ‚Äî the next deploy overwrites them. Edit
the source repo and push; it redeploys.

- A machine ‚Üí `masautt-inc/devices`, `_devices/<slug>.md`
- A hotkey ‚Üí `masautt-inc/config`, `ubuntu/gnome-shortcuts/media-keys.dconf`

The homepage cards for `/dev` and `/hotkeys` are generated too: `config/site/build.mjs`
emits them and `splice-cards.mjs` writes them into `index.md` between the
`<!-- <name>-card:begin (generated by config/site) -->` markers. The Devices card is
hand-written.

## Standalone device HTML

`https://masautt.github.io/devices/<slug>.standalone.html` ‚Äî a self-contained single-file
copy of each device page, CSS and images inlined. Exported by `masautt-inc/devices` during
its deploy (it used to be generated here, which is why `asus-c101` had none).

## Preview locally

Needs Ruby + Jekyll (`gem install bundler jekyll`), then `jekyll serve`
(http://localhost:4000). No Ruby handy (e.g. on the Chromebook)? `./preview.sh`
serves a rough static preview with Python ‚Äî good enough to eyeball layout, but
Liquid tags won't render.
