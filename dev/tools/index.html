Desktop tools

Three Electron apps I actually use daily, built on one shared shell. The interesting part isn't any single tool — it's that the app bar, the theme switcher, the test base and the packaging all live in two versioned packages, and each app inherits them instead of owning them.

localhub

A local GitHub, over the tree.

Points at ~/source/orgs/ and renders it the way a forge would: owners, repos, branches, commits, readmes. It's the tree from the parent page made browsable — no network, no API tokens, nothing to log into. Refresh is explicit, so it never sits there re-scanning a hundred and fifty repos in the background.

Express · React 19 · React Router 7 · Vite · Electron

storybook

A project viewer you can point anywhere.

Opens any project directory and turns what's already in it into something readable: git history as a narrative, and dated design and plan documents laid out on a month calendar, so a repo's paper trail is browsable by when it happened rather than by filename.

React 19 · Vite · Electron · provider hooks per project type

masaudit

What's actually installed, in a table.

Walks the local .NET stack and writes what it finds into SQLite, then serves it as a queryable audit. Storage is node:sqlite from the standard library, so the whole thing runs with zero native dependencies and survives a Node upgrade without a rebuild.

Node 24 · node:sqlite · React 19 · Electron

The shared shell

All three are thin. electron-ui owns the main-process helpers, window and theme persistence, the packaging config and the afterpack step; react-ui owns the app bar, the design tokens and the light/dark switcher. Both are published as private GitHub Packages and consumed by version — so a fix to the app bar ships as a version bump in three apps, and none of them can quietly fork it.

That's the same idea as the layout on the parent page, one level up: decide the shape once, write it down somewhere executable, and let every consumer inherit it.
